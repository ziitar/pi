## Design Decisions
- Token estimation: ~4 chars per token (matching existing compaction system in pi-agent-core)
- Compression threshold: 80% of contextLimit (configurable)
- Context limit: "auto" resolves from model.contextWindow, or explicit number override
- No real LLM calls, no tokenizer library imports (simple char-based estimation)
- Uses pi-ai Message type directly for interop
